## Training Phase — Real-time Set Analytics + Critical Validation Before Logging (critical)

**Problem:** LLM blindly accepts whatever the user reports and immediately logs it without any critical analysis. This leads to phantom sets, wrong weights, misattributed exercises, and corrupted training history.
Confirmed in logs: 27.5 kg logged while previous 3 sets were 10 kg — accepted without question.

### Background validation (silent, runs on every set before logging)

The LLM must run these checks before deciding to log:

1. **Weight anomaly** — is the weight >50% higher or lower than the previous set of the same exercise? If yes → do not log, ask to confirm
2. **Rep anomaly** — did reps drop >50% vs previous set without any user comment about fatigue? If yes → flag and ask
3. **Exercise ambiguity** — is the message ambiguous about which exercise this belongs to? (e.g. mid-transition between exercises, or user just says "done a set") → ask which exercise before logging
4. **Typo / unit confusion** — did user likely mean kg but wrote reps (or vice versa)? e.g. "100 reps bench" or "2 kg" for a heavy compound → ask to confirm
5. **Set number mismatch** — does user's stated set number not match DB progress + 1? → silently correct using DB, note the discrepancy in response
6. **Continuation vs new exercise** — if user reports a set and it's unclear whether it's another set of the current exercise or the start of a new one → ask explicitly: "Is this a continuation of [X] or the start of [Y]?"

If any check fails → **do not log** until user confirms. Ask one clear, specific clarifying question.

### Set analysis feedback (in every response after logging)

Every response after a set is logged must include a brief critical analysis of that set:
- Compare to previous sets in this session: weight trend, rep trend, effort comment
- Compare to same exercise in last session: is the user above/below their previous level?
- Flag if something looks off even after logging: "Logged, but weight dropped 40% vs previous set — is everything ok?"
- Never respond with just a confirmation. The analysis IS the response.

Format:
```
[Set X logged: Y kg x Z reps]
[Analysis: how this compares to previous sets and last session]
[Next set recommendation: weight or rep target]
Ready?
```
